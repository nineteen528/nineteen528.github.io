JavaScript之原型

Some tips for javascript

proto && prototype

任何普通对象和函数对象都具有proto这个属性，它指向它上层构造函数的原型，即prototype，但是普通对象是没有prototype属性的,只有函数对象才会有这个属性，所有构造函数的pototype都是Object(除了Function.prototype,它是function)

所谓的原型链其实就是上面的proto属相所串联起来的东西，构成原型链的实质是poto而不是原型prototype,prototyp只是在其中起到一个辅助作用，在调用一个对象的属性时，该对象会先查找自己的实例属性，即在自身里面查找，如果找不到，再去自己的proto属性里根据原型链一级一级的往上找，直到null。

所有构造器或者函数的proto都指向Function.prototype,它是一个空函数(Function.prototype除外，Function.prototype指向Object.potytype)

constructor

一个对象的constructor属性指的是它的构造函数的prototype.constructor,一般情况下，每个函数都有protoype这个东西，这个东西是个对象，它包含一个constructor属性，指向的就是函数自己。js的本意应该是让每个对象的constructor都指向自己的构造函数，但是这个本意却可以被人们肆意的修改。
