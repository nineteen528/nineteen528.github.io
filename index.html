浮动的框可以向左或向右移动，直到它的外边缘碰到包含框或另一个浮动框的边框为止，由于浮动框不在文档的普通流中，所以文档的普通流中的块框表现得就像浮动框不存在一样。先看向右浮动的框

再请看下图，当框 1 向左浮动时，它脱离文档流并且向左移动，直到它的左边缘碰到包含框的左边缘。因为它不再处于文档流中，所以它不占据空间，实际上覆盖住了框 2，使框 2 从视图中消失。

到这里为止貌似都很正常，但注意到一个细节，框1被设置为float之后，框2被框1给覆盖了，按照w3c的解释，貌似也说得通，那么问题来了，同样是块级元素，为什么p没有被覆盖呢，也就是说，为什么p标签中的文字能围绕被设置为float的图像周围而不被图像覆盖呢？

后来查了不少博客，终于发现一个比较合理的解释：脱离文档流，也就是将元素从普通的布局排版中拿走，其他盒子在定位的时候，会当做脱离文档流的元素不存在而进行定位。需要注意的是，使用float脱离文档流时，其他盒子会无视这个元素，但其他盒子内的文本依然会为这个元素让出位置，环绕在周围。

这样一来可以解释图像为什么不会覆盖掉文字了，其实只要这个块级元素里面有东西（文本或图像。。。），该块级元素本身会被覆盖，但是里面的内容则会被浮动的元素“挤”出来，形成了所谓的环绕（个人观点，欢迎指正）。

标准CSS盒模型

盒子模型有两种，分别为标准盒子模型和IE盒子模型，这里我们忽略IE盒子模型，只讲述标准盒子模型，因为浏览器默认情况下会使用自身的怪异模式来解析渲染页面，所以我们在编写网页时，可以在第一行加上一句<!DOCTYPE html>，意思是让浏览器用标准模式解析页面。

下面正式开始介绍css标准盒模型。

元素框的最内部分是实际的内容，直接包围内容的是内边距，内边距的边缘是边框，边框以外是外边距，外边距是透明的，因此不会遮挡其后的任何元素。

内边距、边框和外边距都是可选的，默认值是零。但是，许多元素将由用户代理样式表（浏览器内置样式表）设置外边距和内边距。可以通过将元素的 margin 和 padding 设置为零来覆盖这些浏览器样式。

在 CSS 中，width 和 height 指的是内容区域的宽度和高度。增加内边距、边框和外边距不会影响内容区域的尺寸，但是会增加元素框的总尺寸。

JavaScript之原型

Some tips for javascript

proto && prototype

任何普通对象和函数对象都具有proto这个属性，它指向它上层构造函数的原型，即prototype，但是普通对象是没有prototype属性的,只有函数对象才会有这个属性，所有构造函数的pototype都是Object(除了Function.prototype,它是function)

所谓的原型链其实就是上面的proto属相所串联起来的东西，构成原型链的实质是poto而不是原型prototype,prototyp只是在其中起到一个辅助作用，在调用一个对象的属性时，该对象会先查找自己的实例属性，即在自身里面查找，如果找不到，再去自己的proto属性里根据原型链一级一级的往上找，直到null。

所有构造器或者函数的proto都指向Function.prototype,它是一个空函数(Function.prototype除外，Function.prototype指向Object.potytype)

constructor

一个对象的constructor属性指的是它的构造函数的prototype.constructor,一般情况下，每个函数都有protoype这个东西，这个东西是个对象，它包含一个constructor属性，指向的就是函数自己。js的本意应该是让每个对象的constructor都指向自己的构造函数，但是这个本意却可以被人们肆意的修改。
